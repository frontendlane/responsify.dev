Responsify

Responsify generates a CSS calc() value that smoothens the "jump" in element's width, height, padding, top etc. caused by a @media query.

Example

  1. By default <div> element expands to fill the width of its container.

    div {
    width: auto; /* default */
}

  2. Suppose the <div> element needs to be 30% of its parent width on resolutions ‚â§ 1000px.

    div {
    width: auto; /* default */
}

@media (max-width: 1000px) {
    div {
        width: 30%;
    }
}

    The @media query causes <div> size to "jump" at 1000px.

  3. To smoothen the transition we add another @media query that gradually changes the <div> width from 100% to 30%.

    div {
    width: auto; /* default */
}

@media (max-width: 1000px) {
    div {
        width: 30%;
    }
}

@media (min-width: 1001px) and (max-width: 1200px) {
    div {
        width: calc(-4200px + 450%);
    }
}

    The width value comes from a simple but annoying-to-calculate equation. Responsify simplifies and automates the calculation.
