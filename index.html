Responsify

Responsify generates a CSS calc() value that smoothens the "jump" in an element's width, height, padding, top etc. caused by a @media query.

Example

  1. By default <div> element expands to fill the width of its container.

    div {
    width: auto; /* default */
}

  2. Suppose the <div> element needs to be 30% of its parent width on resolutions â‰¤ 1000px.

    div {
    width: auto; /* default */
}

@media (max-width: 1000px) {
    div {
        width: 30%;
    }
}

    The @media query causes <div> size to "jump" at 1000px.

  3. To smoothen the transition we add another @media query that gradually changes the <div> width from 100% to 30%.

    div {
    width: auto; /* default */
}

@media (max-width: 1000px) {
    div {
        width: 30%;
    }
}

@media (min-width: 1001px) and (max-width: 1200px) {
    div {
        width: calc(-4200px + 450%);
    }
}

    The width value comes from a simple but annoying-to-calculate equation. Responsify simplifies and automates the calculation.

Tweaking generated value later is hard because input values aren't preserved. So it's a good practice to add a comment documenting them.

When you generate a calc() value with the form above, a comment is added alongside it in your clipboard. Should you decide to use your own comment format, make sure to specify the units as well. Below is an example of what would be copied to your clipboard.

width: calc(9.286px + 4.911vw); /* viewport lower bound: 320px; viewport upper bound: 1440px; element lower bound: 25px; element upper bound: 80px; */

Sass @function

Sass @function

You can use Responsify as a Sass @function. Just download __responsify.scss where your root Sass file is (look for style.scss) and add @import "responsify"; at the top of it. If you use Dart Sass, consider using the recommended @use "responsify"; instead.

The added bonus of using Sass @function is that you don't need comments.

Sass usage examples

Example 1

Responsify <div>'s padding from 40px at 320px viewport width to 90px at 1024px viewport width. Note that calc() will be applied at all viewport widths, not just between 320px and 1024px.
div {
    padding: responsify(320, 1024, 40, 90, "px");
}
CSS output
div {
    padding: calc(-2232.72727px + 7.10227vw);
}

Example 2

Responsify <img>'s height from 60% at 320px viewport width to 30% at 1024px viewport width. Again the calc() will be applied at all viewport widths.
img {
    height: responsify(320, 1024, 60, 30, "%");
}
CSS output
img {
    height: calc(-5044.36364px + 16.36364%);
}

Example 3

Responsify <section>'s margin-top from 50px at 320px viewport width and 100px at 1000px viewport width. In this example calc() will be applied up to and including 1000px viewport width. Note the omitted optional "px" argument at the end.
@media (max-width: 1000px) {
    section {
        margin-top: responsify(320, 1000, 50, 100);
    }
}
CSS output
@media (max-width: 1000px) {
    section {
        margin-top: calc(-2302.94118px + 7.35294vw);
    }
}

Example 4

Responsify <video>'s width from 100vw to 60vw, between 320px and 1024px viewport. In this example width is lower bound at 300px and upper bound at 1800px via clamp() function.
video {
    width: clamp(300px, responsify(320, 1024, 100, 60, "vw"), 1800px);
}
CSS output
video {
    width: clamp(300px, calc(-13061.81818px + 41.81818vw), 1800px);
}
